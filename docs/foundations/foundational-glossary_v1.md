## Canonical Terms
- reason: explicit execution/closure rationale.
- missing_condition: unmet prerequisite for gate acceptance.
- closure: state transition to completed gate result.
- gate: validation boundary controlling transitions.
- invariant: condition that must always hold.
- threshold: numeric/logic boundary in acceptance logic.
- admissible: allowed to enter next processing boundary.
- interpretable: semantically coherent for downstream use.
- merge-ready: authorized for controlled integration.
